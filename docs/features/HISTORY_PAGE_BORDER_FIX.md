# 历史记录页面边框显示修复

## 🐛 问题描述

历史记录页面在默认状态下边框不显示的问题。

## 🔧 修复内容

### 1. **移除CSS变量依赖**
- 将所有CSS变量（如 `var(--border-color)`）替换为具体的颜色值
- 确保样式在所有环境下都能正确显示

### 2. **明确边框样式**
所有主要元素都添加了明确的边框定义：

#### 搜索框区域
```css
.search-box {
  border: 1px solid #e2e8f0;  /* 添加明确边框 */
  background: #ffffff;
  /* ... 其他样式 */
}
```

#### 输入框
```css
.search-input {
  border: 2px solid #e2e8f0;  /* 明确边框颜色 */
  background: #ffffff;
  color: #2d3748;
  /* ... 其他样式 */
}
```

#### 搜索选项按钮
```css
.search-option {
  border: 1px solid #e2e8f0;  /* 添加边框 */
  background: #f7fafc;
  /* ... 其他样式 */
}
```

#### 搜索按钮
```css
.search-btn {
  border: 1px solid #4a6cf7;  /* 添加边框 */
  background: #4a6cf7;
  /* ... 其他样式 */
}
```

#### 结果项
```css
.result-item {
  border: 1px solid #e2e8f0;  /* 添加边框 */
  background: #ffffff;
  /* ... 其他样式 */
}
```

#### 消息气泡
```css
.message {
  border: 1px solid #e2e8f0;  /* 添加边框 */
  /* ... 其他样式 */
}

.message.user {
  border-color: #bbdefb;  /* 用户消息特殊边框色 */
}
```

### 3. **增强交互效果**

#### 悬停效果
- 搜索框悬停时边框变为主色调
- 结果项悬停时边框高亮
- 按钮悬停时有明确的视觉反馈

#### 焦点状态
- 输入框获得焦点时有明确的边框和阴影效果

### 4. **颜色值标准化**

使用的标准颜色值：
- 主色调：`#4a6cf7`
- 主色调悬停：`#3a5ce4`
- 主要文本：`#2d3748`
- 次要文本：`#718096`
- 背景色：`#ffffff`
- 次要背景：`#f7fafc`
- 边框色：`#e2e8f0`
- 用户消息背景：`#e3f2fd`
- 用户消息边框：`#bbdefb`

## 📝 技术细节

### CSS变量替换
- 所有 `var(--color-name)` 都被替换为具体颜色值
- 确保在没有CSS变量定义的环境下也能正常显示

### 边框策略
- 使用 `1px solid` 作为标准边框
- 输入框使用 `2px solid` 以增强视觉重要性
- 悬停状态改变边框颜色而非添加边框，避免布局跳动

### 兼容性
- 移除了对CSS变量的依赖
- 使用标准CSS属性确保广泛兼容性
- 保持了原有的响应式设计
